# 提交规范

本页面说明实验代码的提交规范和验收要求。

## 代码提交

### Git 使用规范

1. **提交信息规范**
   ```bash
   git commit -m "[P1] 完成任务1：引导块制作"
   git commit -m "[P2] 实现进程切换功能"
   ```

2. **分支管理**
   - main 分支保持稳定
   - 每个实验建议创建新分支
   - 完成后合并到 main

3. **提交频率**
   - 完成一个小功能就提交
   - 不要等全部完成才提交
   - 便于回溯和查找问题

### 代码风格

- 使用一致的缩进（建议 4 空格）
- 函数和变量命名要有意义
- 适当添加注释
- 删除调试用的 printf（除非必要）

## 实验报告

每个实验完成后需要提交实验报告，包括：

1. **实验目的和要求**
2. **实验原理和设计**
3. **实验步骤和结果**
4. **问题和解决方案**
5. **实验总结和心得**

## 验收流程

1. **提前预约** - 向助教预约验收时间
2. **代码演示** - 在 QEMU 或开发板上演示功能
3. **代码讲解** - 解释关键代码的实现
4. **问题回答** - 回答助教的相关问题

## 评分标准

| 项目 | 占比 | 说明 |
|------|------|------|
| 功能完整性 | 50% | 是否完成所有要求的功能 |
| 代码质量 | 30% | 代码规范、可读性、健壮性 |
| 理解程度 | 20% | 对实现原理的理解深度 |

## 注意事项

!!! warning "学术诚信"
    - 禁止抄袭他人代码
    - 禁止直接使用网上的完整解决方案
    - 可以参考资料，但要理解并自己实现
    - 违反学术诚信将严肃处理

!!! info "截止时间"
    - 请在规定时间内完成并验收
    - 特殊情况请提前联系助教
    - 逾期提交可能影响成绩
