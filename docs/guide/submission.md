# 提交规范

本页面说明实验代码的提交规范和验收要求。

## 代码提交

### Git 使用规范

1. **提交信息规范**
   - 以简洁明了的形式描述你干了什么

   ```bash
   git commit -m "[P1-task1] finish the bootloader"
   ```
   - 了解 commit message 的严格规范请参考[这里](https://chris.beams.io/posts/git-commit/)

2. **分支管理**
    - 每个实验都要创建新分支
    - 创建后拉取 startcode 

3. **提交频率**
    - 完成一个 task 就提交
    - 不要等全部完成才提交
    - 便于回溯和查找问题

### 代码风格

- 使用一致的缩进（建议 4 空格）
- 函数和变量命名要有意义
- 适当添加注释

## 验收流程

1. **预订顺序** - 提前确定验收顺序
2. **代码演示** - 在开发板上(O2 参数编译)演示功能
3. **代码讲解** - 解释关键代码的实现
4. **问题回答** - 回答助教的相关问题

## 注意事项

!!! warning "学术诚信"
    - 禁止抄袭他人代码
    - 禁止直接使用网上的完整解决方案
    - 可以参考资料，但要理解并自己实现
    - 违反学术诚信将严肃处理
